보험 도메인 데이터 사이언티스트. 7년차. 한국.

한국 외국계 생명보험사에서 2년쯤 일하고 있다. 인수심사 자동화(XGBoost), 청구 사기 탐지(NN), 설계사 가동 예측(LightGBM), 통계 기반 추가가입 모형까지 네 개를 운영한다. 인수심사 모형이 GA 25회차 유지율을 +7.6%p 올렸고, 사기 탐지는 작년에 22억 정도 방어했다. 가명처리 솔루션은 기획부터 도입까지 끌었다 — 기법 15개, 적정성 평가 프로세스 포함. AI 거버넌스(RUAI)랑 임직원 AI 교육 프로그램도 같이 운영하고, AI 기본법·금융분야 AI 가이드라인 같은 거 D&T에 발표하거나 금감원 CPC 대응도 한다.

그전엔 NLP 스타트업 검색개발팀 매니저로 5년 반. BM25 + Contrastive Learning IR부터 MRC, RAG 파이프라인까지. 약관 RAG QA 만들 때 할루시네이션 90% 줄이고 정확도 30% 올린 게 기억에 남는다. 손해보험, 증권, 보증 쪽 금융권 레퍼런스도 좀 있다. SFT, DPO, LoRA 파인튜닝도 거기서 다 만져봤다.

요즘은 모바일 페북 스크린샷을 옵시디언 노트로 자동 변환하는 파이프라인을 만들고 있다. 사진첩 EXIF 시간으로 그룹핑하고, OCR이나 Vision LLM으로 텍스트 뽑고, URL 있으면 trafilatura로 본문 가져오고, Qwen3.5-122B가 정리한다. 같이 돌아가는 게 두 개 더 있다 — 신모델 출시되면 자동 감지해서 내 도메인 100문제로 평가하는 하니스, 그리고 KIRI나 금감원·금융위 발표 보고서에 응답하는 글 시리즈. 여기 [Lab](/lab/)에 결과 일부가 쌓이고 있다.

도구는 Python · SQL 기본에 XGBoost · LightGBM · PyTorch · HuggingFace · BERT/ELECTRA, LLM 쪽 OpenAI · Claude · Qwen, LangChain/LlamaIndex/Milvus/Elasticsearch, Azure ML/Databricks, 로컬은 MLX. ADsP. OPIc IH라 글로벌 본사 협의 가능.
